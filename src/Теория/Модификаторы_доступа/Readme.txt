Уровень доступа или др.словами - модификаторы доступа:

public - доступен из других классов
protected - доступен внутри класса/пакета/в производных классах даже если они из разных пакетов
default - по-умолчанию. Доступно всем классам внутри пакета
private - доступен только внутри класса


Модификатор без доступа - Static/non static

1 уровень класса (static) - к таким полям/методам можно обращатся по имени класса/по созданному эксземпляру класса
    - Когда описываются переменные внутри класса, по-умолчанию, для каждого объекта создается новая копия этой переменной.
    Если указать static, то для каждого статического объекта переменная существует в одном эксземпляре

    - внутри static метода нельзя использоваться НЕ static переменные/константы. И наоборот - в не static методе нельзя использовать static переменные

    - переменная класса НЕ static, то каждый объект класса, будет иметь свое знач. этой переменной
    - переменная static, то каждый объект класса, будет иметь одно знач. этой переменой
    https://javarush.ru/quests/lectures/questsyntaxpro.level11.lecture01

    - статические методы невозможно переопределять (Override), можно только "прятать" (hidden)
    - статическое поле может быть с любым модиф.доступа
        - для статического поля не "красиво" писать гетторы => приватные статические поля тоже уродливы
